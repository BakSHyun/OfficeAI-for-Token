# OfficeAI 사용 설명서

## OfficeAI란?

한 문장으로 업무를 지시하면 AI 에이전트 팀이 작업을 나누어 실행하고, 필요한 경우에만 사용자 승인을 요청하는 데스크톱 오케스트레이션 앱입니다. 토큰·비용은 작업 난이도에 맞는 모델 티어로 자동 배정됩니다.

## 빠른 시작

1. 앱을 설치하고 실행합니다.
2. 온보딩에서 API 키를 입력하거나 **데모(mock)** 로 둘러봅니다.
3. 상단 CommandBar에 업무를 입력합니다. 예: `이번 주 마케팅 브리핑 초안 작성해줘`
4. 대시보드 **3D 오피스**에서 에이전트(고양이)가 일하는 모습을 확인합니다.
5. **승인 대기** 탭에서 계획·예산·검토 결과를 승인/반려합니다.
6. **보고서** 탭에서 산출물을 확인하고 복사·MD 저장합니다.

## 주요 화면

| 화면 | 설명 |
|------|------|
| 대시보드 | 3D/2D 오피스, 실시간 토큰·비용, 절약률 |
| 업무 관리 | 과거 run 목록, 정기 업무 예약 |
| 승인 대기 | 결정이 필요한 요청 인박스 |
| 보고서 | 완료된 run의 산출물·절약 리포트 |
| 에이전트 | 2D DAG 프로세스 뷰 |
| 설정 | 모델 티어, API 키, 예산, 업무 기억, 라이선스 |

## 모델 티어

| 티어 | 용도 |
|------|------|
| local | Codex/Cursor Agent CLI 등 로컬·구독 기반 |
| economy | 저비용 API 호출 |
| standard | 일반 업무 |
| premium | 고난이도·고위험 작업 |

설정에서 티어별 provider와 모델을 변경할 수 있습니다. 개발·테스트 시에는 전 티어를 `mock`으로 두면 API 키 없이 동작합니다.

### LM Studio로 무료·오프라인 실행

인터넷·API 키 없이 내 PC의 로컬 모델로 돌리고 싶다면 [LM Studio](https://lmstudio.ai)를 씁니다.

1. LM Studio에서 모델을 받고 **Local Server**를 시작합니다 (기본 주소 `http://localhost:1234/v1`).
2. OfficeAI 설정 → 모델 티어에서 원하는 티어의 provider를 **lmstudio**로 바꾸고, model에 LM Studio에 로드된 모델 이름을 입력합니다.
3. API 키는 필요 없습니다. 저장 후 명령을 내리면 로컬 모델이 응답합니다.

비용이 0으로 잡히므로 토큰 절약 리포트에서도 로컬 실행이 반영됩니다.

## 업무 기억 (로컬 맥락)

설정 → **업무 기록 폴더 연결**에서 Markdown·Cursor 기록 등이 있는 폴더를 선택하면, 이후 명령에 해당 맥락이 자동 포함됩니다. 데이터는 로컬에서만 처리됩니다.

## 정기 업무

업무 관리 → **정기 업무**에서 요일·시각과 명령을 등록하면 앱이 백그라운드에서 자동 실행합니다. 결과는 OS 알림으로 알려줍니다.

## 3D 오피스

- **3D / 2D** 토글로 뷰 전환
- **추적**: 선택한 캐릭터를 카메라가 따라감
- 설정에서 완료/실패 사운드 on/off (기본 꺼짐)

## 라이선스·체험

- **체험**: mock 티어 무제한, OpenAI/Anthropic 등 유료 API 티어는 10회
- **정식**: 설정에 라이선스 키(`OAIV1....`) 입력 후 활성화
- 키 발급은 판매 채널을 통해 받습니다 (개발용: `npx tsx scripts/issue-license.ts`)

## 문제 해결

- **이벤트가 안 옴**: `npm run app:dev`로 Electron 실행 여부 확인
- **API 오류**: 설정에서 키·티어 provider 확인
- **진단**: 설정 → 진단 파일보내기 (시크릿 자동 제거)
- 자세한 개발·배포: `docs/dev-guide.md`, `docs/release-guide.md`

## 단축 팁

- CommandBar **↑/↓**: 최근 명령 탐색
- 캐릭터 클릭 → 우측 TaskInspector에서 담당 에이전트 확인
- 예산 한도·환율은 설정 → 예산에서 조정 (₩ 표시)
